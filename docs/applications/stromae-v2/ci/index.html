CI

Stromae V2 is hosted on github so we use Github Actions for continuous integration. The CI is divided into 4 files according to the actions, although I think a single file with conditions would be more readable.

Github Secrets​

If you fork the repository you will need to enable actions and to add some secrets.

Secret NameExplication
DOCKERHUB_REGISTRY_HOSTThis is the registry host of the docker hub repository (usually the same as username but could change if your using bot )
DOCKERHUB_USERNAMEDocker hub username of the account which publish image
DOCKERHUB_TOKENThe docker hub token linked with username
GITHUB_TOKENSee official docs

The inseefr/stromae projet publish docker image in the inseefr/stromae dockerhub projet.

Push on Branches named v2-* (except v2-develop and v2-master)​

Each branch named v2-* will trigger a build and a docker stage at every push. See docker.yml file for more. The docker tag will be the name of the branch.

Pull request on v2-*​

Every pull request targeted on branch named v2-* will trigger a build stage. See here for more.

Push on branch v2-develop​

Every push on v2-develop will trigger a build, release and docker stage. Unfortunately, if the version has not been updated you will encounter an error because release stage will try to make a github release that already exist. In v2-develop, release is a release candidate ("version-in-package.json"-rc) and same for docker tag. See here for more.

Push on branch v2-master​

This is the same as v2-develop except that it is not release candidate but real with "version-in-package.json". See here for more.
